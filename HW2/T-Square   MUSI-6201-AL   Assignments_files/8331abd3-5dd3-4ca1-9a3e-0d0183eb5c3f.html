For the following questions (Q1 - Q3), generate two test signals: x1(t) = A*cos(2*pi*f*t), let {A = 0.5, f = 441 for 0 <= t < 1} and {A = 0.5, f = 882 for 1 <= t <= 2}), x2(t) = A*cos(2*pi*f*t), let {A = 0.5, f = 441 for 0 <= t < 1} and {A = 0.9, f = 441 for 1 <= t <= 2}). 
 
1. [15] Novelty function: spectral flux (textbook p.44, eq.3.30)
  • Implement a matlab function:  [nvt] = mySpectralFlux(x, windowSize, hopSize) that computes the block-wise spectral flux from the audio signal.     
  • For the above mentioned test signals, compute their novelty functions and plot the results. Based on the results, locate the onset position (in second) and discuss the wether or not this novelty function is suitable for the test signals, why or why not?.
  • Hints: diff( )
 
2. [15] Novelty function: peak envelope (textbook p76, eq.4.9)
  • Implement a matlab function:  [nvt] = myPeakEnv(x, windowSize, hopSize) that computes the block-wise peak envelope from the audio signal.     
  • For the above mentioned test signals, compute their novelty functions and plot the results. Based on the results, locate the onset position (in second) and discuss the wether or not this novelty function is suitable for the test signals, why or why not?.
 
3. [15] Novelty function: weighted phase deviation (attached paper, section 2.4) 
  • Implement a matlab function:  [nvt] = myWPD(x, windowSize, hopSize) that computes the block-wise weighted phase deviation from the audio signal.
  • For the above mentioned test signals, compute their novelty functions and plot the results. Based on the results, locate the onset position (in second) and discuss the wether or not this novelty function is suitable for the test signals, why or why not?.    
  • Hints: spectrogram( ), angle( ), unwrap( )
  • [4] Bonus: Implement your phase wrap and unwrap function and explain your method in the report. 
 
For the following questions (Q4 - 6), please use the attached training data. 
 
4. [15] Adaptive threshold: median filter (textbook p128, eq.6.15)
  • Implement a matlab function: [thres] = myMedianThres(nvt, order, lambda) that computes the signal adaptive median threshold. 
  • Read ‘2-artificial.wav’ in the training data (please see attached files), use any of the above novelty functions, compute the median threshold (for the first 3 seconds) and plot the threshold (in green) on top of the novelty function (in red). Please generate three figures with increasing “order” and another three figures with increasing “lambda”. Discuss the general trend by manipulating these parameters. What could be the cause of the “delay” in the adaptive threshold? How could we compensate this delay?
  • Hints: median( )
 
5. [15] Onset detection with adaptive thresholding
  • Implement a matlab function: [onsetTimeInSec] = myOnsetDetection(nvt, fs, windowSize, hopSize) that calls your myMedianThres( ) function, apply the median thresholding, and detect the onsets. 
  • For test signal x1(t) and x2(t), please select a suitable novelty function and use your thresholding function to detect the onset. Plot the novelty functions (in red) and their corresponding thresholds (in green). Please specify and discuss your selection of novelty function, parameters, and verify the results. 
  • Hints: findpeaks( ) 
 
6. [15] Standard evaluation metrics: precision, recall and f-measure (textbook p.131, eq.6.19-6.21) 
  • Implement a matlab function: [precision, recall, fmeasure] = evaluateOnsets(onsetTimeInSec, annotation, deltaTime) that compares the detected onsets to the annotation and calculates the statistical evaluation measures. 
  • To verify your implementation, you may use testEval.mat with deltaTime set to 50. You should get the precision = 0.50, recall = 0.10, fmeasure = 0.1667. In addition, verify your implementation with test = [ ] or ref = [ ], and your function should return 0 for all measures. 
 
7. [10] Overall evaluation
  • Report the average precision, average recall and average f-measure of the given training set. Specify and discuss your selection of adaptive thresholding parameters.  
  • A reference result using windowSize = 1024, hopSize = 256, deltaTime = 50 (ms) is shown in the table. Please use the same parameters for your evaluation.
  • Your f-measures shouldn’t be 10% less than the reference implementation. Note that your functions will be tested on another testing set for final grading. (chose your parameters wisely)
